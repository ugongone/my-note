質問タイプに応じた検索語追加

ユーザーの「曖昧な言葉」をドキュメントの「正確な言葉」へ。
インテント（意図）理解による検索クエリ最適化の仕組み。

なぜこれが必要なのか？（言葉のズレ問題）

ユーザーの入力

「経費精算」

（面倒だから単語しか打たない）

ドキュメントのタイトル
  • 経費精算システムログ
  • 経費精算規定（定義）
  • 経費精算マニュアル（手順）

ユーザーは「やり方」を知りたくても、「経費精算」としか入力しません。そのままだと、システムは「定義」や「ログ」など関係ない資料もヒットさせてしまいます。この「言葉のズレ」を埋めるのが本機能です。

処理の仕組み（2ステップ）

1

インテント判定

LLMや分類器が、ユーザーの質問が「何を知りたがっているか」を分類します。

入力: "VPN接続できない"
判定: Trouble型 (解決策)
2

検索語(キーワード)注入

判定結果に基づき、ドキュメントでよく使われる「書き言葉」を強制的に追加します。

ルール: Trouble型なら...
+ "対処法", "原因", "エラー"

最適化された検索

拡張されたクエリで検索を実行。

"VPN接続" AND
("対処法" OR "原因" OR "エラー")

質問タイプ別：追加されるマジックワード

How型 (手順)

頻出

「やりたい」「操作したい」という意図。

追加キーワード

手順 方法 マニュアル ステップ

ターゲット文書: マニュアル、手順書、操作ガイド

What型 (概要)

「意味を知りたい」「全体像」という意図。

追加キーワード

とは 概要 定義 Introduction

ターゲット文書: 規定集(第1条)、製品概要資料

Trouble型 (解決)

「困っている」「直したい」という意図。

追加キーワード

原因 対処法 解決策 エラーコード

ターゲット文書: FAQ、障害報告書、トラブル集

効果シミュレーション：「PC 初期化」と検索した場合

Before: 通常検索 Query: "PC 初期化"
  • PC初期化時の情報漏洩リスク規定
    ...初期化はリスクを伴うため...
  • 2023年度 PC廃棄一覧表
    ...全台初期化済み...

本当に欲しい「手順」が出てこない...

After: How型として処理 Query: "PC 初期化" + 手順 OR 方法
  • Windows PC 初期化手順書.pdf
    ...1. 設定を開く 2. 回復を選択...
  • PCセットアップマニュアル（管理者用）
    ...第5章：トラブル時の初期化方法...

ピンポイントで手順書がヒット！

運用のポイント：設定の外出し管理

コードに直書きしない

キーワードをプログラムの中に直接書いてしまうと、単語を1つ追加するたびにシステムの更新（再デプロイ）が必要になります。実務では設定ファイル（JSON等）として分離し、柔軟に管理することが推奨されます。

メリット：

  • PDCAが高速化： 検索ログを見て「ヒットしない単語」があれば即座に追加設定が可能。
  • 誰でも管理可能： エンジニアでなくても、管理画面からJSONを編集するだけで調整ができる。
intent_config.json の例
{
  "intent_rules": [
    {
      "id": "how_to",
      "description": "手順・方法を知りたい意図",
      "inject_keywords": ["手順", "方法", "マニュアル", "ステップ"],
      "boost_score": 1.5
    },
    {
      "id": "trouble",
      "description": "トラブル解決の意図",
      "inject_keywords": ["原因", "対処法", "解決策", "エラー"],
      "boost_score": 2.0
    }
  ]
}

技術的な肝： 企業ドキュメントの「お作法（テンプレート）」を逆手に取ることで精度を高めています。
