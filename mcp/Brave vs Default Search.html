標準検索 vs Brave Search MCP

2025年現在、Claude Codeには標準で検索機能が搭載されていますが、なぜ多くの開発者が追加でBrave Search MCPを導入するのか？その決定的な違いを解説します。

比較項目
標準検索機能
Brave Search MCP
プライバシー・追跡
低～中
Google等のエンジンを使用するため、検索クエリやIPが追跡される可能性がある。企業ポリシーによってはNG。
非常に高い
ユーザー追跡を行わない方針。社内コードに関連するキーワードを含めても比較的安全。
トークン効率 (AIの記憶容量)
低い (重い)
HTMLページ全体や広告などのノイズ情報も読み込みがちで、コンテキストウィンドウを圧迫しやすい。
高い (軽い)
API経由で「構造化データ（JSON）」として必要な情報だけを返すため、AIが処理しやすく、トークン消費が少ない。
技術情報の精度
一般的
一般的なウェブ検索結果。SEO対策された記事や古い情報が上位に来ることもある。
開発者寄り
技術ドキュメントやReddit、StackOverflowなどのフォーラム情報に対するインデックス精度が高く評価されている。
導入の手間
ゼロ
Claude Codeをインストールした時点で即座に使用可能。
要設定
Brave APIキーの取得（無料枠あり）と設定ファイルの記述が必要。

標準検索機能で十分な場合

  • ライブラリのバージョン番号をちょっと確認したい
  • 一般的な用語の意味を調べたい
  • セットアップの手間をかけたくない

Brave Search MCPを入れるべき場合

  • トークン制限ギリギリの長文コードを扱っている
  • 仕事で使っており、検索ログを残したくない
  • 標準検索がエラーやハルシネーションを起こした時の予備（バックアップ）として
  • 古い記事を除外し、最新のドキュメントだけを厳密に探させたい

結論：併用が最強のソリューション

標準機能を「メイン」として使いつつ、Brave Search MCPを「高精度・高セキュリティなサブウェポン」として導入するのが、2025年のパワーユーザーの標準的な構成です。
特に、Brave SearchはAPIを通じて「AIが読みやすい形式」で情報を渡すことに特化しているため、Claudeの思考精度が向上するという報告も多数あります。
