GLM-5.1 — オープンソースがクローズドモデルを超えた日

SWE-Bench Proでオープンソース初のトップスコア。NVIDIA GPUなしで訓練。MITライセンスで商用利用自由。「オープンソースはクローズドに勝てない」という通念を覆したモデルの全貌。

コーディングではトップ、汎用推論・数学ではクローズドモデルが依然優位。ただし差は大幅に縮小しており、GLM-5.1は「特化型」ではなく汎用性を維持しつつコーディングで突破したモデルと言える。

🔓 なぜオープンソースがクローズドを超えにくいのか

1. 計算資源の非対称性

OpenAIやAnthropicは数百億ドル規模でGPUクラスタを構築。オープンソース開発者は通常この規模にアクセスできない。

2. データの質と量

独自のデータパイプラインとRLHFへの膨大な投資。訓練データの質は最終性能に直結する。

3. ポストトレーニングの深さ

事前学習後のファインチューニング・安全性調整・推論最適化に多大な工数。ここでクローズドが差をつけやすい。

🚀 GLM-5.1が突破できた4つの要因

1. 「長時間自律作業」という新しい最適化軸

従来モデルが分単位のインタラクション向けなのに対し、GLM-5.1は数千回のツールコールに渡ってゴール整合性を維持。METR評価で8時間連続作業可能なOSSモデルは初。

2. MoEアーキテクチャの効率性

754B中アクティブ40B。巨大モデルの知識量を持ちつつ推論コストを抑制。1台のH100で推論可能な規模でデプロイのハードルを大幅に低下。

3. プラトー回避の最適化技術

訓練中の性能頭打ち（プラトー効果）を早期に検出し局所最適解から脱出。同じ計算資源からより多くの性能を引き出す。

4. 非NVIDIAハードウェアでの訓練

Huawei Ascend 910B + MindSporeで全訓練を完了。NVIDIAに依存しないことで地政学リスクを回避し、Huaweiチップのフロンティア対応能力を実証。

「エージェントは去年末には約20ステップこなせた。GLM-5.1は現在1,700ステップ実行できる。自律的な作業時間は、スケーリング則の次に最も重要な曲線かもしれない」 — Z.ai リーダー Lou（X投稿）

* Claude Codeでの利用について — 要注意

Z.aiの公式ドキュメントにはClaude Codeのセットアップ手順が掲載されている（環境変数でエンドポイントとモデル名を差し替える方式）。しかしAnthropicが公式にサポートしているプロバイダーはAnthropic API / Bedrock / Vertex AI / Foundryの4つのみ。Claude Code側がGLM-5.1を認識しているわけではなく、Z.aiがAnthropic Messages API互換のエンドポイントを構築して接続を実現している。Anthropicの方針変更により将来使えなくなるリスクがあり、ツール呼び出しの完全な互換性も保証されていない。

ローカルデプロイは vLLM / SGLang / KTransformers をサポート。Hugging Faceからウェイトをダウンロードしてセルフホスト可能。API利用は入力$1.00/出力$4.00（1Mトークン）、ピーク時（14:00-18:00 北京時間）は3倍消費だが4月末までオフピーク1倍のプロモーション中。

速度が遅い理由

前世代GLM-5（89 tok/s）から約40%低下。長時間自律タスク（8時間・1,700ステップ）の精度を優先した設計であり、対話的なチャット速度よりも持久的な推論品質にリソースを配分している。

TTFTは優秀

最初のトークンまでの時間（TTFT）は1.7秒で、GPT-5.4の推論モード（217秒）やClaude Sonnet 4.6 max（147秒）と比べると桁違いに速い。待たされる「体感」は出力速度だけでは測れない。

プロバイダー次第で高速化可能

Z.ai公式では約44 tok/sだが、GMI Cloud（FP8量子化）では133.9 tok/s、FriendliAIでは122.5 tok/sを記録。オープンウェイトの強みとして、推論最適化が進めば大幅な高速化が見込める。

💡 意味するもの

SWE-Bench Proでの0.7ポイント差は統計的議論の余地はある。汎用推論でもクローズドが優位。しかし「OSSはクローズドに追いつけない」という心理的な壁は確実に崩れた。

Z.aiは2026年1月に香港証取に上場、時価総額約528億ドル。フラッグシップをMITで公開しつつGLM-5 Turboをプロプライエタリで提供する「ハイブリッド戦略」で、オープンソースAIがビジネスとして成立することを証明している。
