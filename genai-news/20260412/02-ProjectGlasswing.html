⚠️ なぜ「危険」なのか

コーディング能力とサイバーセキュリティ能力は表裏一体。脆弱性を理解し修正できるモデルは、同時にそれを発見し悪用するポテンシャルも持つ。

「AIモデルは、ソフトウェアの脆弱性を発見・悪用する能力において、最も熟練した人間を除くすべての人間を上回るレベルに到達した」 — Anthropic, Project Glasswing公式ページ

実際にMythos Previewは数週間のテスト期間ですべての主要OSとWebブラウザにおいて数千件のゼロデイ脆弱性を発見済み。これらは既にメンテナに報告されパッチ済み。

🗓️ 一般公開の見通し

Anthropicは「Mythos Previewを一般公開する予定はない」と明言。ただし最終目標はMythosクラスのモデルを安全にスケール展開することだとしている。

「今後リリースするClaude Opusモデルに新しいセーフガードを搭載し、Mythos Previewほどのリスクを伴わないモデルで改善・洗練を進める予定だ」 — Anthropic, Project Glasswing公式ページ

ロードマップ

  1. 現在: Project Glasswingで防御的にMythosを展開し、脆弱性を事前修正
  2. 90日以内: 初期調査結果を公表（発見された脆弱性と緩和策）
  3. 次期Opus: 新しいセーフガードを搭載してMythosクラス以外のモデルで試行
  4. 将来: 安全性が確認されたらMythosクラスのモデルを広く展開

具体的な時期は未定だが、少なくとも数ヶ月〜半年以上のセーフガード開発期間が必要になると考えられる。

🔮 攻防バランスの転換

これまでサイバーセキュリティは「攻撃側が有利」な構造にあった。防御側は全脆弱性を塞ぐ必要があるが、攻撃側は1つ見つければよい。Mythosクラスのモデルはこの非対称性を両方向に加速させる。

Project Glasswingは「防御側に先手を打たせる」ための時間稼ぎ。類似能力のモデルが他社から登場するのは時間の問題であり、その前に重要インフラの脆弱性をできるだけ修正しておく戦略だ。Anthropicが米政府高官に「今年中に大規模サイバー攻撃の可能性を大幅に高める」と非公式に警告したとAxiosが報じている。
